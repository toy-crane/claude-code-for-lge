# 요구사항 설계

`$ARGUMENTS`에 대한 요구사항을 대화로 구체화한 뒤 문서로 정리합니다.

## 1단계: 질문으로 빈칸 채우기

사용자와 대화하며 요구사항의 빈칸을 채웁니다.

### 질문 규칙

- **한 번에 질문 하나.** 절대 두 개 이상 질문하지 마세요.
- `AskUserQuestion`을 사용하고, 2-4개 선택지를 제시하세요.
- 이전 답변을 기반으로 다음 질문을 만드세요.

### 질문 차원

필요한 순서대로 질문합니다. 모든 차원을 다 물어볼 필요는 없습니다.

1. **범위** -- 1차 출시에 포함할 것과 뺄 것
2. **엣지 케이스** -- 예외 상황의 시스템 반응
3. **성공 기준 구체화** -- 모호한 기준을 입력/출력으로
4. **전제 확인** -- 암묵적으로 가정한 기술/환경

### 종료 조건

다음 중 하나면 질문을 멈추고 생성 단계로 이동합니다:

- 사용자가 "됐어", "충분해" 같은 신호를 보냄
- 주요 빈칸이 모두 채워짐
- 3회 이상 질문한 후 새로운 발견이 없음

## 2단계: 문서 생성

대화에서 확인한 내용을 바탕으로 `references/output-format.md`를 읽고 형식에 맞춰 작성하세요.

### 작성 규칙

- 구현 방법(어떤 라이브러리, 어떤 아키텍처)은 쓰지 마세요. "무엇을"에만 집중하세요.
- 모호한 부분은 추측하지 말고 미결정 사항에 나열하세요.
- 성공 기준은 "이 입력이면 이 출력" 형식으로 작성하세요.

## 3단계: 저장

완성된 문서를 `.claude/specs/` 디렉토리에 저장합니다.

- 파일명: `.claude/specs/<feature-name>.md`
- 저장 전 사용자에게 파일명 확인
