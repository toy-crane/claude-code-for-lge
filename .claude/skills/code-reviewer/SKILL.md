# Code Review Skill

## 리뷰 절차

1. 변경된 파일 목록을 확인합니다
2. 각 파일의 diff를 읽고 다음을 점검합니다:
   - 버그 가능성 (null 체크 누락, 경계값 처리)
   - 보안 취약점 (SQL injection, XSS)
   - 성능 문제 (불필요한 루프, N+1 쿼리)
   - 네이밍 컨벤션 준수 여부

3. 발견한 이슈를 심각도별로 분류합니다:
   - MUST FIX: 버그, 보안 문제
   - SHOULD FIX: 성능, 가독성
   - NICE TO HAVE: 스타일, 개선 제안

## 리뷰 출력 형식

### 요약
- 변경 범위 한 줄 요약

### 이슈
- **[MUST FIX]** 파일명:라인 -- 설명
- **[SHOULD FIX]** 파일명:라인 -- 설명

### 긍정적 피드백
- 잘 작성된 부분도 언급
