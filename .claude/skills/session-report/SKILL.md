# Session Report

현재 세션의 대화 내용을 분석하여 `.claude/session/` 디렉토리에 마크다운 보고서를 생성한다.

## 워크플로우

1. 전체 대화 내용을 분석하여 주요 작업, 의사결정, 변경사항을 파악
2. `git status`와 `git diff --stat`으로 실제 변경된 파일 목록 확인
3. `git branch --show-current`로 현재 브랜치 확인
4. [references/template.md](references/template.md) 템플릿을 읽어 보고서 구조 참조
5. `.claude/session/` 디렉토리에 보고서 파일 저장

## 파일명 규칙

`YYYY-MM-DD-핵심키워드.md`

예시:
- `2026-02-25-session-report-스킬-생성.md`
- `2026-02-25-todo-앱-버그-수정.md`

## 작성 지침

- 한글로 작성
- 요약은 2~3문장으로 세션의 전체 맥락을 전달
- 주요 의사결정은 테이블 형식으로 결정/선택/이유를 명확히 기록
- 변경된 파일은 git 상태 기반으로 실제 변경 사항만 포함
- TODO 항목은 구체적이고 실행 가능한 형태로 작성
- 이슈/블로커가 없으면 "없음"으로 표기
